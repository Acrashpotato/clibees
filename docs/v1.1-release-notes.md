# clibees v1.1 发布说明

## 版本概览

- 版本基线：`v1.0.0` -> 当前工作区增量（本次发布以一次性封板方式产出 `v1.1.0`）。
- 目标：把 V1 的冻结设计继续落到可操作界面与 API 行为上，降低 run 级运维与审计路径的摩擦。
- 变更范围（按文件归类）：
  - `src/**`：运行时、控制、UI API、读模型与存储层增强。
  - `apps/console/**`：路由、页面、样式系统、设置与子模块化 UI。
  - 根配置/文档：`README.md`、`AGENTS.md`、`.multi-agent*.yaml`、`package*.json` 等同步更新。

## 核心能力更新

- UI API 契约与服务面升级（`src/ui-api/contracts.ts`、`src/ui-api/server.ts`）：
  - 统一 `projection envelope` / `action envelope`。
  - 新增与固化 run-scoped projection 路由：run list、workspace、task board、task detail、session detail、approval queue、audit timeline、manager chat。
  - 补充分页与过滤语义（approval queue / run list 等）。
- 运行 CLI 选择与执行链路强化：
  - `src/ui-api/selected-cli.ts` 固化 `codex/codefree/claude` 取值集合。
  - `src/config/run-cli-config.ts` 使 run 级 agent/profile 与 selected CLI 对齐。
  - `src/ui-api/terminal-gateway.ts` 新增终端流网关（WebSocket + node-pty），支持 run/session 终端会话流式交互。
- 会话与消息持久化增强：
  - `src/storage/session-store.ts` 引入 run 级消息聚合读取与 message/thread/session 的一致更新时间维护。

## Console/UI 更新

- 信息架构重构为“任务池 + 右侧子菜单内容区”：
  - 任务池右侧承载二级入口：总管、工位池、工作台、执行车道、审批、审计。
  - 点击二级入口后，内容在右侧头部下方铺满，不再切换到全屏独立页面。
- 控制台导航与头部交互优化：
  - 左侧竖向侧栏移除，顶部房子图标回到任务池。
  - 右上角改为齿轮设置入口。
  - 二级子页面头部提供“返回图标”回到二级菜单集合。
- 页面与组件扩展：
  - 新增 `ManagerPage`、`WorkerpollPage`、`SettingsPage`。
  - 审批详情拆分（`pages/approvals/comp/ApprovalDetailPanel.vue`）。
  - 审计视图拆分（`pages/inspect/comp/*`）。
  - 会话详情增强与实时终端相关逻辑（`pages/session-detail/comp/*`）。
- 样式体系升级：
  - 从单文件样式迁移为 `styles/` 分层结构（tokens/base/layout/components/pages + legacy scoped）。

## 运行时与编排更新

- 读模型构建器扩展：
  - 新增 manager chat / workerpoll projection 构建器与索引导出（`src/ui-read-models/index.ts`）。
  - 现有 workspace/task/session/approval/audit 相关 projection 构建器持续补齐字段与一致性映射。
- 协调与控制相关增强：
  - `run-coordinator`、entrypoint、adapter/config 相关模块对 run-scoped 元数据与执行路径做了对齐更新。
  - `delegated-manager` 覆盖场景测试补充（`src/control/delegated-manager.test.ts`）。

## 配置与工程化更新

- 根配置与运行配置更新：
  - `.multi-agent.yaml`、`.multi-agent.allow-outside.yaml` 对 run 创建参数与工作区写入策略做了联动补充。
  - `package.json` / lockfile 与 console 子工程依赖同步。
- 控制台设置能力落地：
  - 提供 run 默认 CLI、自动启动、审批筛选/刷新、审计默认 run、workspace 轮询等本地偏好配置。

## 公共接口/类型变化

- UI API（外部消费面）
  - `CreateRunRequest` 新增/明确：`cli`、`allowOutsideWorkspaceWrites`、`autoResume`、`configPath`。
  - 新增 projection 路由定义与统一 envelope 输出规范。
  - 终端流接口通过 `TerminalGateway` 暴露 `terminalSessionId + wsPath` 进行会话连接。
- Console 路由（外部导航语义）
  - 以 `/runs` 为任务池主入口，run-scoped 子页面统一为 `/runs/:runId/*`。
  - 二级页面在任务池右侧内容区内渲染，保留任务上下文与头部操作。

## 兼容性与迁移说明

- 兼容保留：
  - 既有 run/task/session 核心语义保持不变，UI API 主路径仍保持 `/api/runs/*` 主体模式。
- 行为变化：
  - Console 导航从“侧栏主导”转为“头部 + 任务池右侧承载子页面”。
  - 审批页部分文案与组件已拆分并修正编码问题，界面语义更清晰。
- 建议：
  - 如有自定义前端脚本直接依赖旧页面布局，应改为依赖路由与 projection 数据，不绑定旧 DOM 结构。

  - `README.md` 与本发布说明已对齐 v1.1 结论与术语。
